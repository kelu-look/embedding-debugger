## How to update

1. Pull fresh GitHub metrics into section 1 with a new row (don't
   overwrite older rows — keep a growth trail).
2. Add new rows to the relevant section as items come in.
3. When in doubt, archive the link (Wayback Machine) so the evidence
   survives.
4. For each new entry, prefer a one-line summary and an external link
   over pasting long text inline.
